# 命名与结构约定

## 占位符规范

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{Module}` | 模块实体类名前缀，大驼峰 | `KmNews` |
| `{module}` | 模块实体名，小驼峰 | `kmNews` |
| `{模块Key}` | 模块标识，小写+连字符 | `km-news` |
| `{modulePrefix}` | 模块包名前缀 | `km` |
| `{moduleSuffix}` | 模块包名后缀 | `news` |

---

## ref 文件命名规则

```
ref-{序号}-{语义描述}-{后缀}.md
```

| 后缀 | 适用场景 | 示例 |
|------|---------|------|
| `-spec` | 机制能力型（分类、附件、审批等） | `ref-01-category-spec.md` |
| `-integration` | 引擎集成型（表单引擎、流程引擎等） | `ref-01-form-integration.md` |

序号 `00` 固定为 `bundle-spec`（全局包导入规范），业务 ref 从 `01` 起。

---

## ref 拆分原则

| 情况 | 拆分方式 |
|------|---------|
| 有 Entity 定义或调整 | 以 Entity 为单位组织一个 ref，覆盖 Entity → VO → API → Repository → Service → Controller 全栈 |
| 无 Entity | 独立能力单独一个 ref（工具类 / 枚举 / 配置 / i18n） |

---

## ref 文件内部结构

每个代码单元按以下格式组织：

```
### 步骤N：{单元名称}

**核心要点**：
- 要点说明
- 关键表格（继承关系、接口约定等）

​```java
// 代码块（含行内注释，说明"为什么"）
​```

**注意事项**：
1. 易错点一
2. 易错点二
```

## SKILL.md 章节结构

| 章节 | 是否必须 |
|------|---------|
| frontmatter（name + description） | 必须 |
| 一、实体概览 | 有多种实体类型时必须 |
| 二、部署流程（含 Maven + 执行表格） | 必须 |
| 三、部署验证清单 | 必须 |
| 四、代码注释规范 | 必须 |
| 五、占位符说明 | 必须 |
