# 机制类型定义与分类说明

## 机制分类

| 类型 | 说明 | ref 后缀 | 典型示例 |
|------|------|---------|---------|
| **机制能力**（Spec） | 框架提供的独立机制，以 Entity + 全栈形式部署 | `-spec` | 分类机制、附件机制、审批机制 |
| **引擎集成**（Integration） | 需对接外部引擎，以集成适配为主 | `-integration` | 表单引擎、流程引擎、消息引擎 |

---

## ref 拆分原则

### 有 Entity 定义或调整时
以 Entity 为切入点，一个 ref 文件覆盖该 Entity 的完整全栈：

```
Entity → VO → API → Repository → Service → Controller
```

### 无 Entity 时
独立能力单独一个 ref 文件：

```
工具类 / 枚举 / 配置 / i18n
```

---

## 文件命名规则

```
ref-{序号}-{语义描述}-{后缀}.md
```

| 后缀 | 适用场景 | 示例 |
|------|---------|------|
| `-spec` | 机制能力型 | `ref-01-category-spec.md` |
| `-integration` | 引擎集成型 | `ref-01-form-integration.md` |
